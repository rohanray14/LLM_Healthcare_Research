{% if not has_creds %}

To enable Google Sheets integration, set these environment variables on Render:

GOOGLE_SHEETS_CREDENTIALS_JSON The full JSON content of your Google service account key file
GOOGLE_SHEET_ID The spreadsheet ID from the Google Sheet URL
e.g. from https://docs.google.com/spreadsheets/d/SHEET_ID_HERE/edit
Setup Steps
  1. Go to Google Cloud Console and enable the Google Sheets API
  2. Go to IAM & Admin > Service Accounts and create a service account
  3. Create a JSON key for the service account and download it
  4. Share your Google Sheet with the service account email (the client_email in the JSON)
  5. On Render, add GOOGLE_SHEETS_CREDENTIALS_JSON with the full JSON content and GOOGLE_SHEET_ID with your sheet ID
  6. Redeploy the app

Push Annotations to Sheet

Export all annotations, claim spans, and expert reviews to the Google Sheet. This replaces existing data in the sync tabs.

This also happens automatically after every annotation save.

Pull Input Data from Sheet

Import new posts and comments from the first tab of the sheet. Only adds posts that don't already exist in the database.

Expected columns: post_id, title, body, label1, label2, label3, num_comments, reddit_url, comment_1, comment_2, ...

Re-sync from Excel Source Files

Re-read the Excel source files and update the database with any changes. This updates existing posts (labels, text, comments) and adds any new posts. Annotations are preserved.

Use this after editing the Excel files directly to ensure the database and Google Sheet stay in sync across all contributors.
